🎮 [플래티넘 퀘스트] 기능명세서
안녕하세요! 이번 과제에서 요구된 필수 구현 요소들을 중심으로, 제가 어떤 기능을 어떤 방식으로 고민해서 구현했는지 정리했습니다. 전체적으로 코드가 너무 뚱뚱해지지 않도록, 기능별로 스크립트를 나누어(SRP 원칙 참고) 구현하려고 노력했습니다.

1. 플레이어 입력 및 이동 (Input & Movement)
어떤 기능을: 키보드(WASD)와 마우스 입력을 받아 캐릭터를 이동시키고 시점을 회전하며, 걷기와 달리기 속도를 인스펙터에서 쉽게 조절할 수 있도록 만들었습니다.

어떤 방법으로: Unity의 Input System 패키지와 Rigidbody의 linearVelocity를 활용했습니다.

어떻게 구현했는지: 입력을 받는 역할만 전담하는 PlayerInputReader 스크립트를 따로 만들어서 여기서 WASD와 마우스 값을 읽어오도록 했습니다. 이 값을 CharacterMotor3D 스크립트가 넘겨받아 이동 처리를 합니다.
이동은 Update가 아닌 FixedUpdate에서 물리 기반(Rigidbody)으로 처리했고, 카메라가 바라보는 방향을 기준으로 캐릭터가 움직이도록 방향 벡터를 계산했습니다. 걷기 속도와 뛰기 속도(Shift 키)는 하드코딩하지 않고 [SerializeField]를 사용해 유니티 에디터 상에서 변수로 뺐습니다.

2. 중력 및 점프 (Gravity & Single Jump)
어떤 기능을: 무한 점프를 방지하는 단일 점프를 구현하고, 자연스러운 중력을 적용했습니다. 점프 높이 역시 에디터에서 조절 가능하게 열어두었습니다.

어떤 방법으로: Physics.CheckSphere를 이용한 바닥 판정(Ground Check)과 Rigidbody.AddForce를 활용했습니다.

어떻게 구현했는지:
바닥 판정도 별도의 GroundChecker 컴포넌트로 분리했습니다. 캐릭터 발밑에 빈 오브젝트(Check Point)를 두고 지정된 반경 안에서 'Ground' 레이어와 충돌하는지 검사합니다.
바닥에 닿아있을 때만(IsGrounded == true) 점프 입력이 먹히도록 해서 이중 점프를 막았습니다. 점프할 때는 기존 Y축 속도를 0으로 초기화한 뒤 ForceMode.Impulse로 위쪽으로 힘을 줘서 항상 일정한 높이로 점프할 수 있게 했습니다. 또한, 공중에 있을 때는 자체적으로 중력 계수(Gravity Multiplier)를 곱해 조금 더 빠르게 떨어지도록 해서 점프가 붕 뜨는 느낌을 줄였습니다.

3. 애니메이션 처리 (Animation & Blending)
어떤 기능을: 가만히 있을 때(Idle), 걸을 때(Walk), 뛸 때(Run)의 상태가 뚝뚝 끊기지 않고 자연스럽게 이어지도록 구현했습니다. (추가 구현 요소인 블렌딩 기능 포함)

어떤 방법으로: Animator의 Blend Tree와 파라미터 보간(Damp) 기능을 활용했습니다.

어떻게 구현했는지:
애니메이터에 Idle, Walk, Run을 각각 두지 않고 하나의 1D Blend Tree로 묶었습니다. 캐릭터의 현재 이동 상태에 따라 0(정지), 0.5(걷기), 1(뛰기)의 값을 애니메이터에 넘겨주는데, 이때 SetFloat의 dampTime을 사용하여 수치가 서서히 변하게 만들었습니다. 덕분에 키보드를 누르자마자 확 뛰지 않고 부드럽게 모션이 전환됩니다. 점프 애니메이션은 Trigger를 사용해 점프 시작 시점에 재생되도록 이벤트 방식으로 연결했습니다.

4. 카메라 설정 (3rd Person Camera)
어떤 기능을: 3인칭 시점(TPS)에서 마우스를 움직여 캐릭터 주변을 둘러보고, 캐릭터의 이동에 맞춰 카메라가 자연스럽게 쫓아가는 카메라를 구현했습니다.

어떤 방법으로: Unity의 Cinemachine (시네머신) 패키지를 활용했습니다.

어떻게 구현했는지:
직접 카메라 추적 로직을 짜는 것보다 시네머신을 쓰는 것이 퀄리티가 좋다고 판단했습니다. 씬에 Cinemachine Camera를 배치하고, 캐릭터 목 부분에 타겟용 빈 오브젝트(LookPos)를 두어 카메라가 거리를 유지하며 따라다니도록 세팅했습니다.
마우스 움직임에 따라 시점을 자유롭게 돌리기 위해 시네머신 내장 기능인 Input Axis Controller를 연결하여, 마우스 델타 값이 카메라의 공전(Orbit) 수치로 반영되도록 구현했습니다
